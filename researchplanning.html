研究テーマと問い

  • テーマ：インドネシアにおける**近リアルタイム森林減少アラート（GFW Integrated Alerts）**は、森林減少を抑制するか。
  • 主たる問い：アラート頻度が高い地域ほど、その後の**年次森林損失（ha / 率）**が小さくなるか。
  • 貢献：ブラジルDETERの証拠（Assunção et al. 2023 など）をインドネシアに拡張。**雲量を操作変数（IV）**に用いた識別で、アラート“可視性”の外生変動を利用する。

データと単位

  • 空間×時間単位：GAUL2015の ADM2（県・市）× 年。主に 2019–最新年（アラートの利用が安定）。
  • 主要変数
  • アウトカム：Hansen GFC v1.12（2001–2024）
  • loss_ha：年損失ha
  • defor_rate = loss_ha / forest2000_ha（基準：treecover2000 ≥ 30%）
  • 処置：GFW Integrated Deforestation Alerts（GLAD-L/S2/RADD統合）
  • ADM2×年で件数／面積相当を集計（信頼度しきい値でマスク）
  • IV（操作変数）：雲量
  • 1次案：Sentinel-2 Cloud Probability をADM2×年平均の雲被覆シェアに集計
  • 代替（バックアップ）：MODIS Global Cloud Cover Frequency（約1km）
  • 統制：
  • 降水量：CHIRPS（年合計、雨季合計〔前年11月–当年3月〕）
  • 火災：MCD64A1（焼失面積 burned_ha）、VIIRS（火点数 fire_cnt・FRP合計）
  • 気候指数：ENSO（ONI 年平均）
  • （任意）夜間光（VIIRS NTL）、人口、保護区比率、アクセス時間 等
  • 取得方法：Google Earth Engine（GEE）で全国を集計 → CSV を書き出し
  • 大容量処理は島別×年範囲に分割、tileScale・simplify() で省メモリ化
  • R / Stata で ADM2×year のマスターパネルに結合（キー：ADM2_CODE, year）

識別戦略（概念）

  • 内生性：アラートは「伐採が起きやすい場所・時期」と相関する恐れ → IV推定
  • IVの論理：雲量↑ → 衛星の観測可能性↓ → 検出アラート↓
  • 関連性（relevance）：雲とアラートは強い負相関が期待
  • 排除制約（exclusion）：降水や火災等の天候コントロールを入れたうえで、雲そのものは伐採決定には直接影響しない、と想定（頑健性で検証）

回帰式（2SLS：ADM2・年の固定効果）

第一段階
[
\text{alerts}{i,t}
= \alpha+\beta_1,\text{cloud}{i,t}
+\beta_2,\text{rain}{i,t}
+\beta_3,\text{fire}{i,t}
+\mu_i+\lambda_t+\varepsilon_{i,t},
]
第二段階
[
\text{defor}{i,t}
= \gamma+\delta,\widehat{\text{alerts}}{i,t}
+\theta^\top X_{i,t}
+\mu_i+\lambda_t+\upsilon_{i,t}.
]

  • (i)：ADM2、(t)：年
  • (X_{i,t})：ENSO、雨季降水、焼失面積（または火点/FRP）等
  • 推定：クラスタSE（ADM2単位）。必要に応じて Conley（空間HAC）標準誤差も報告。

進捗（取得済み／進行中）

取得済み

  • GFC（年損失・基準森林面積）、GFWアラート（2019–）、CHIRPS（年合計・雨季合計）、MCD64A1（burned_ha）、VIIRS火点/FRP（Hotspot）、夜間光（VIIRS NTL）
  • R 側でフォルダ構成・マスターキー統一の準備済み（ADM2_CODE, year で結合）

進行中

  • 雲量（IV）：Sentinel-2 Cloud Probability を ADM2×年平均に集計（バックアップにMODIS雲頻度）

期待される結果（仮説）

  • 第一段階：(\beta_1<0)（雲↑ → alerts↓）、KP F 統計は十分大
  • 第二段階：(\delta<0)（alerts が森林損失を抑制）
  • 異質性：市場アクセスや保護区シェア、行政能力の高い地域で効果が大きい可能性

主要な頑健性チェック案

  1. 雲量の定義・解像度（S2 1km vs Landsat 30m、集計期間の変更）
  2. 火災の指標（焼失面積 vs 火点/FRP）
  3. 天候データの代替（CHIRPS ↔ IMERG/ERA5-Land）
  4. 目的変数の仕様（loss_ha ↔ defor_rate、対数化、winsorize）
  5. FE の強化（ADM2×島・年×島 など）／サンプル除去（極小離島・大都市）
  6. 経済活動の統制（夜間光・人口）と相互作用（メカニズム探索）
